# UI Guidelines

## 方向性
- iOS ネイティブ感
- 白ベース
- 余白多め
- Widget 風カード UI
- 長時間見ても疲れない配色

## デザイン原則
- 情報を詰め込みすぎない
- 1画面で俯瞰できる構成を優先
- 操作導線は最短（タップ数を減らす）
- 夫婦で共有しやすい柔らかいトーン

## カラーパレット（初期）
- Primary: `#5C6BC0`
- Surface: `#FFFFFF`
- Background: `#F6F7FB`
- Text Primary: `#1D2736`
- Text Secondary: `#455468`
- Border: `#E6EAF2`

## レイアウト
- 8pt グリッド
- カード角丸は 12〜16
- 主要セクション間余白は 16〜24

## タイポグラフィ
- タイトル: 700
- 本文: 400
- 最小フォントサイズ: 14

## 実装指針
- コンポーネントは小さく分割可能な構造で作る
- 一度に多機能化せず、MVP → 改善の順で進める
